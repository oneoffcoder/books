Collection initialization
-------------------------

.. highlight:: python
   :linenothreshold: 1

When initializing empty Python collections such as lists and dictionaries, favor using the corresponding literal delimiters instead of the corresponding functions.

Don't do this
^^^^^^^^^^^^^

.. code:: python

    names = list() # initialize list
    data = dict()  # initialize dictionary

Do this
^^^^^^^

.. code:: python

    names = [] # initialize list
    data = {}  # initialize dictionary


Using the literal delimiters ``[]`` and ``{}`` is faster and more Pythonic than using ``list()`` and ``dict()``. The literal form is faster because the generated bytecode does not need to look up a function name.

You can inspect the bytecode for :code:`[]` as follows.

.. code:: python

    from dis import dis

    dis('[]')


The bytecode looks like this. Notice that there are only two lines.

.. code:: text

    1           0 BUILD_LIST               0
                2 RETURN_VALUE

You can inspect the bytecode for :code:`list()` as follows.

.. code:: python

    from dis import dis

    dis('list()')

The bytecode looks like this. Notice that there are three lines.

.. code:: text

    1           0 LOAD_NAME                0 (list)
                2 CALL_FUNCTION            0
                4 RETURN_VALUE

Here's the bytecode for :code:`{}`.

.. code:: text

    1           0 BUILD_MAP                0
                2 RETURN_VALUE

Here's the bytecode for :code:`dict()`.

.. code:: text

    1           0 LOAD_NAME                0 (dict)
                2 CALL_FUNCTION            0
                4 RETURN_VALUE
